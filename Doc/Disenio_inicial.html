🚦 Diseño de Arquitectura: Proyecto Travesía

Documentación Técnica y Lógica de Negocio

Introducción del Diseño:
Toda la implementación de los módulos va a estar implementada por la Página de Travesía. Los embebidos en moodle van a estar diseñados a partir de iframes específicos generados por nuestra página que van a estar divididos de la siguiente manera:
  • Iframe Dashboard Dinámico
  • Iframe Formulario
  • Iframe Configuración Formulario
  • Iframe Ingesta de Datos
  • Iframe Semáforo

Implementación de Módulos

El sistema opera de forma paralela a Moodle, centralizando la lógica en el servidor de Travesía.

Arquitectura de la Lógica de Negocio

1. Base de Datos y Controladores

Base de Datos: Almacenamiento local para entidades y registros históricos.

Endpoints (Controladores):

  • GET: Recuperación de estados de semáforos y perfiles.
  • POST: Carga masiva de CSVs y respuestas de formularios.
  • PUT/PATCH: Modificación de parámetros de configuración.

Entidades: Profesor, Alumno Inicial, Alumno Avanzado.

2. Modelos de Procesamiento

  • Cálculo de Semáforos: Motor de reglas para procesar riesgos.
  • Dashboard Dinámico: Agregación de datos (compatible con exteriorización a Looker).
  • Modelo de Formulario: Lógica de generación dinámica y validación de completación.
  • Limpieza de Datos: Módulo de saneamiento para inputs de CSV y formularios.
